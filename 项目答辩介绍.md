# 基于EAIDK-310的人脸识别系统设计与实现 - 答辩材料

## 1. 项目概况
- **课题名称**: 基于EAIDK-310人脸识别系统设计与实现
- **核心目标**: 在嵌入式平台(EAIDK-310)上实现实时、准确的人脸检测与跟踪。
- **主要功能**: 实时视频采集、多种人脸检测算法切换、人脸跟踪、GUI可视化交互。

## 2. 技术方案
### 2.1 硬件环境
- **平台**: EAIDK-310 (基于RK3228H, ARM Cortex-A53) / PC (开发调试)
- **输入设备**: USB摄像头
- **显示设备**: HDMI显示器 / 远程桌面

### 2.2 软件架构
- **开发语言**: Python 3.8+
- **核心库**: OpenCV (图像处理), PyTorch/Ultralytics (深度学习), PyQt5 (图形界面)
- **架构模式**: 模块化设计 (MVC思想)，分离界面(GUI)、逻辑(Detectors)和配置(Config)。

## 3. 核心算法实现
本项目集成了三种不同量级的检测算法，以适应不同的应用场景：

### 3.1 OpenCV Haar级联分类器 (传统方法)
- **原理**: 基于Haar特征和Adaboost级联分类器。
- **特点**: 速度极快，CPU占用低，但对光照、角度敏感，误检率相对较高。
- **适用场景**: 算力受限的极端环境，简单正脸检测。

### 3.2 YOLO11 (深度学习 - 高精度)
- **原理**: 基于最新的YOLO (You Only Look Once) 架构，单阶段目标检测。
- **特点**: 检测精度极高，鲁棒性强，能处理遮挡、侧脸等复杂情况。
- **适用场景**: 对准确率要求高的场景，PC端演示。

### 3.3 Yolo-FastestV2 (深度学习 - 轻量级)
- **原理**: 针对移动端优化的轻量级YOLO模型，使用ShuffleNet等轻量骨干网络。
- **特点**: 模型体积极小(KB级别)，推理速度快，精度优于Haar。
- **适用场景**: EAIDK-310等嵌入式设备的实时检测。

### 3.4 人脸跟踪 (IoU Tracker)
- **原理**: 基于检测框的IoU (交并比) 匹配算法。
- **流程**: 检测 -> 计算IoU -> 匈牙利算法匹配 -> 轨迹更新。
- **优势**: 解决了检测抖动问题，实现了ID保持。

## 4. 系统功能展示
- **GUI界面**: 使用PyQt5构建，包含视频显示区、算法选择下拉框、FPS/检测数实时统计、运行日志。
- **多线程设计**: 采用 `QThread` 将视频采集与算法推理放入后台线程，防止界面卡死。
- **配置管理**: 使用 `config.yaml` 统一管理模型路径、阈值、摄像头参数，便于调优。

## 5. 关键代码解析 (答辩备用)
- **多态设计**: 所有检测器继承自统一接口或遵循鸭子类型 (detect方法)，便于扩展。
- **异常处理**: 针对模型加载失败、摄像头无法打开等情况做了完善的异常捕获和日志记录。

## 6. 性能对比 (实验结果)
| 算法 | 速度 (FPS) | 精度 | 资源占用 |
| :--- | :--- | :--- | :--- |
| Haar Cascade | 高 (>30) | 低 | 低 |
| YOLO11 | 低/中 | 高 | 高 (需GPU加速) |
| Yolo-FastestV2 | 中/高 | 中 | 中 |

## 7. 常见答辩问题准备
1. **为什么选择这三种算法？**
   - 答：为了对比传统机器学习与现代深度学习的差异，同时兼顾精度(YOLO11)和嵌入式部署的速度需求(FastestV2)。
2. **如何在EAIDK-310上优化性能？**
   - 答：使用了轻量级模型(FastestV2)，降低了输入图像分辨率(如320x240)，使用了多线程分离UI和计算。
3. **人脸跟踪是如何实现的？**
   - 答：利用前后帧检测框的重叠面积(IoU)进行关联，如果IoU大于阈值则认为是同一个人，并分配相同ID。
